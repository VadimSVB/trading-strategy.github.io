ТВХ от уровня стопового объёма

Выход из баланса вверх

  1. Выход из боковика через "Стоповый объём"
  2. Выход из "Стопового объёма"
  3. Откатное движение обратно к "Стоповому" желательно не импульсым движением
  4. Верхняя граница стопового - будет ТВХ; Нижняя граница "стопового" будет - СТОПОМ"
  5. Гарантированный тейк на обновлении МАX уровня

Выход из баланса вниз

  1. Выход из боковика через "Стоповый объём"

  3. Откатное движение обратно к "Стоповому" желательно не импульсым движением
  4. Нижняя граница стопового - ТВХ; Верхняя граница "стопового" - СТОП"

ТВХ от уровня перепозиционирования со стоповым объёмом

Выход из баланса вверх

  2. Выход из "Стопового объёма"
  3. Откатное движение обратно к уровню перепозицианирования
  4. Верхняя граница ближайшей ОС - ТВХ; Нижняя граница этой ОС - СТОП"
  5. Гарантированный тейк на обновлении МАX уровня

Выход из баланса вниз

  1. Выход из боковика четко выраженным "Стоповый объёмом"
  2. Выход из "Стопового объёма"
  3. Откатное движение обратно к уровню перепозицианирования
  4. Нижняя граница ближайшей ОС - ТВХ; Верхняя граница этой ОС - СТОП"
  5. Гарантированный тейк на обновлении МIN уровня

ТВХ от уровня перепозиционирования без топового объёма

Выход из баланса вверх

  1. Выход из боковика без четко выраженного "Стопового объёма"
  2. Откатное движение обратно к уровню перепозицианирования (без четкого даунтренда)
  3. Верхняя граница ближайшей ОС (скорее всего пружины) - ТВХ; Нижняя граница этой ОС - СТОП"
  4. Гарантированный тейк на обновлении МАX уровня (можно выйти в безубыток , а остальной частью сопровождать импульс)

Выход из баланса вниз

  1. Выход из боковика четко выраженным "Стоповый объёмом"
  2. Выход из "Стопового объёма"
  3. Откатное движение обратно к уровню перепозицианирования (без четкого аптренда)
  4. Нижняя граница ближайшей ОС (скорее всего пружины) - ТВХ; Верхняя граница этой ОС - СТОП"
  5. Гарантированный тейк на обновлении МIN уровня (можно выйти в безубыток , а остальной частью сопровождать импульс)

ТВХ в импульсе - "Агрессивный участник"

Выход из баланса вверх

  1. Выход из боковика четко выраженым "Стоповым объёмом"
  2. Развитие импульса, образование новых ОС цена в которых не доходит в откатах до предидущих ОС. Большие разрывы между ОС, нет ретестов.
  3. Фиксируем появление сильного участника который обеспечит ТВХ на ВГ проторговки. СТОП - ВГ предъидущей проторговки.
  4. Гарантированный тейк на обновлении МАX уровня

Выход из баланса вниз

  1. Выход из боковика четко выраженым "Стоповым объёмом"
  2. Развитие импульса, образование новых ОС цена в которых не доходит в откатах до предидущих ОС. Большие разрывы между ОС, нет ретестов.
  3. Фиксируем появление сильного участника который обеспечит ТВХ на НГ проторговки. СТОП - НГ предъидущей проторговки.
  4. Гарантированный тейк на обновлении МIN уровня

ТВХ в импульсе - "Сильный участник"

Выход из баланса вверх

  1. Выход из боковика четко выраженым "Стоповым объёмом"
  2. Развитие импульса, образование новых ОС цена в которых не доходит в откатах до предидущих ОС. Большие разрывы между ОС, есть ретест с возможным погружением ретестов в зоны предъидущих ОС.
  3. ТВХ - верхняя граница ОС ретеста или центр ОС в которую погрузился ретест. СТОП - нижняя граница этого же накопления
  4. Гарантированный тейк на обновлении МАX уровня

Выход из баланса вниз

  1. Выход из боковика четко выраженым "Стоповым объёмом"
  2. Развитие импульса, образование новых ОС цена в которых не доходит в откатах до предидущих ОС. Большие разрывы между ОС, есть ретест с возможным погружением ретестов в зоны предъидущих ОС.
  3. ТВХ - нижняя граница ОС ретеста или центр ОС в которую погрузился ретест. СТОП - верхняя граница этого же накопления
  4. Гарантированный тейк на обновлении МIN уровня

  1. Выход из баланса, развитие импульса, формирование ОС
  2. Неудачный тест первого трендового уровня без признаков силы с противоположной стороны
  3. Пробитие одной ОС
  4. ТВХ - верхняя граница ОС. СТОП - нижняя граница этой ОС.
  5. При входе в продолжающийся (затухающем) тренде (импульсе) прибыль фиксируется на гарантированном уровне ближайшем Максимуме.

ТВХ - пробой одной ОС. Сброс лишних пассажиров.

Выход из баланса вверх

  1. Выход из боковика четко выраженым "Стоповым объёмом"
  2. Развитие импульса, образование новых ОС по ходу движения.
  3. Появление агрессивного фикса (пробитие ближайшей ОС формированной по ходу движения, импульсом в противоход, бз особого повышения объемов на пике импульса и без создания ОС на этом пике)
  4. ТВХ - верхняя граница ОС. СТОП - нижняя граница этой ОС.
  5. Гарантированный тейк основание проторговки при ближайшем Максимуме. Выходить лучше полностью.

Выход из баланса вниз

  1. Выход из боковика четко выраженым "Стоповым объёмом"
  2. Развитие импульса, образование новых ОС по ходу движения.
  3. Появление агрессивного фикса (пробитие ближайшей ОС формированной по ходу движения, импульсом в противоход, бз особого повышения объемов на пике импульса и без создания ОС на этом пике)
